代申诉开放接口与商家确认 — 精简方案

产品方案　2026-07-15 ｜ 本期只交付开放能力：服务商调接口代填申诉 → 商家微信内确认 → 确认后才正式提交。服务商平台可视化专区不在本期。　接口先行精简稿　关联：完整方案「服务商违规申诉代办」

1. 背景、问题与范围

服务商已在帮商家写申诉、凑材料，但提交仍多靠线下催商家自己点。本期开放「代申诉」能力：服务商把说明和材料交给平台，平台生成确认链接；商家在手机微信里看完再决定是否提交。服务商不能替商家直接点提交。

成功指标：代申诉链路可被第三方平台接入并跑通「发起 → 商家确认/拒绝/超时 → 正式进审或退回」闭环。

本期边界见附录 A。

表 B：业务规则

规则 条件 系统行为
谁能代申诉小程序已授权服务商「基础管理 + 违规申诉协助」两类权限不满足则不可发起
哪些违规可代申诉处罚类可申诉记录（功能限制、账号限制、页面封禁、下架等）警告类不进入本链路
服务商只能代填调用接口成功只生成待确认任务与链接，不直接进审
商家确认后才提交商家在二次确认弹层点「确认提交」进入平台审核队列，扣减 1 次申诉机会
确认前不扣次已发起但未确认、拒绝、超时不消耗申诉次数
确认有效期发起后 24 小时内超时自动失效；双方提交入口恢复
待确认期间服务商已发起、商家尚未确认/拒绝/超时冻结该违规的商家自申诉；服务商不可改材料、不可再发起；不拦截查看违规或其他违规的处理
审核中互斥已有审核中的申诉商家与服务商均不可再提交该违规的申诉
不可撤回已发起确认确认任务已生成只能等确认、拒绝或超时；不可改材料、不可撤销任务
可再次发起拒绝或超时之后次数不限；须重新提交材料并再走确认
结果怎么同步给服务商商家确认 / 拒绝 / 超时，以及后续审核结果主动推送：推到第三方平台事先配置的消息接收地址；查询回补：服务商也可主动查询申诉状态，防止漏收推送
推送与查询回补（白话）：第三方平台在开平后台配过一个「收消息的网址」。商家一点确认或拒绝，微信就往这个网址推一条结果，服务商系统自动收到。若推送偶发没收到，服务商再调查询接口把状态补齐——两套一起用，不是二选一。

3. 页面与交互（商家确认）

同一确认链接按任务状态打开不同页；下列字段均只读。

3.1 全页共用

  • 路径 / 入口：服务商转发的确认链接；须在手机微信内打开
  • 权限：仅小程序管理员、运营者、其他单独开通违规处理权限的人员可查看并操作；非授权身份拦截

3.2 字段与小程序管理后台对照

确认页「申诉内容」区字段对齐设计稿；与商家在小程序管理后台（MP）看违规 / 申诉时的信息对应关系如下（名称以本页展示名为准）：

本页字段 展示规则 与 MP 的关系
违规描述一行或短文，说明违规与处理结果对应 MP 违规详情中的被处理原因+处理内容
处罚对象小程序头像 + 名称对应被处罚的小程序账号
处罚时间日期对应该条处罚下发时间
申诉理由多行文本，服务商代填对应 MP 申诉表单中的申诉说明 / 申诉原因；本页不可改
补充证明材料缩略图网格，可点预览对应 MP 申诉表单中的证明材料；本页不可增删改

ps. MP 后台申诉填写页是可编辑的；本链路是确认页，只读展示服务商已填内容。

3.3 状态与页面

A. 待商家确认

设计稿已有。商家首次打开未处理完的确认链接时展示。

区域 文案 / 内容
页头标题服务商申请为你的「{小程序名}」小程序提交违规申诉
时效说明{截止时间}前未提交自动终止当前申诉（截止时间 = 发起后 24 小时）
申诉内容见 3.2 五个字段，只读
底部按钮暂不提交 / 提交申诉（均不直接生效，先进入二次确认）

本态底部不单独铺长说明；扣次与剩余次数放进二次确认弹层。

B. 二次确认（弹层）

设计稿商家侧目前从主按钮直达结果页，设计稿疑似漏做二次确认；弹层样式可复用设计体系已有确认弹层。

触发 弹层标题 说明文案（弹层中部） 按钮
点「提交申诉」确认提交申诉？确认后将按服务商代填内容正式提交平台审核，并扣减 1 次申诉机会（当前剩余 {n} 次）。取消 / 确认提交
点「暂不提交」确定暂不提交？确定后本次代办申诉不会提交，也不扣减申诉机会（当前剩余 {n} 次）。取消 / 确认拒绝
  • 取消：关闭弹层，回到待商家确认页
  • 确认提交：扣 1 次并正式进审 → 进入「已提交结果」
  • 确定拒绝：不提交、不扣次 → 进入「商家已拒绝」

C. 已提交结果

1）结果过渡页（提交成功瞬间）

区域 文案 / 内容
状态图标成功勾选
主文案已提交「{小程序名}」小程序的{违规摘要}违规申诉
次数说明剩余{m}次申诉机会（m = 扣次后的剩余次数）
底部按钮我知道了（进入审核中详情页）

2）申诉审核中详情页（同链接再次打开，或点「我知道了」后进入）

区域 文案 / 内容
状态标题申诉审核中
次数说明还剩{m}次申诉机会
申诉内容同 3.2 五个字段，只读
底部按钮无操作按钮

D. 商家已拒绝

1）结果过渡页

区域 文案 / 内容
主文案已拒绝提交「{小程序名}」小程序的{违规摘要}违规申诉
次数说明可重新发起申诉，剩余{n}次申诉机会（n 未扣次）
底部按钮我知道了

2）详情页（同链接再次打开）

区域 文案 / 内容
状态标题已拒绝提交申诉
次数说明还剩{n}次申诉机会
申诉内容同 3.2，只读
底部按钮无；服务商需重新发起后才会再有可操作确认页

E. 已超时

设计稿已有。超过截止时间仍未确认时，打开链接进入此态。

区域 文案 / 内容
状态标题申诉未提交
说明超过24小时未提交，可重新发起申诉
申诉内容同 3.2，只读
底部按钮无

不扣次；服务商可重新发起。

3.4 异常

情况 表现
非授权身份拦截，不可查看完整材料与操作
链接无效 / 任务不存在提示确认已失效

附录 A：本期边界

  • 本期上线
    • 代申诉开放接口：提交申诉说明与证明材料，返回商家确认链接
    • 商家确认页（待确认 / 二次确认 / 已提交 / 已拒绝 / 已超时）
    • 确认/拒绝/超时与审核结果：推送到第三方平台；也支持主动查询补齐
    • 确认链接由服务商用自有渠道转发（群消息、客服、自有站内信等）；平台不负责送到商家手机
  • 二期及以后
    • 服务商平台监管专区 UI
    • MP 站内信触达；视情况评估模板消息与管理后台「待确认」入口（见 决策记录/商家确认入口·H5与MP方案对比.md）
  • 不在本期
    • 服务商平台「监管专区」列表、弹窗、控制台
    • 平台代商家直接提交（跳过确认）
    • 模板消息作为商家确认主触达；小程序管理后台「待确认」页签
    • 确认发起后的撤回、修改；确认后的撤销

附录 D：术语

术语 含义

第三方平台服务商在开放平台侧承接授权与接口调用的账号体系；本期对接主体
消息接收地址第三方平台在开平配置的网址，用于接收微信推送的业务结果
查询回补推送之外，服务商主动查状态，把漏掉或延迟的结果补齐
待商家确认已代填并发起确认，但尚未进审、尚未扣次
二次确认商家点主按钮后弹出的确认层；说明文案在弹层中部，须含是否扣次与剩余次数
